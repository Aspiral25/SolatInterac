Admin Panel

The Admin Panel (AdminDialog) provides a password-protected interface for surau administrators to configure, test, and manage all aspects of Solat Interactive.

🔒 Security Note: The default admin password is 123. Change it immediately after the first launch to prevent unauthorized access.

Opening the Admin Panel

From the main display, click the Admin button (hidden in fullscreen mode — press F11 to reveal it). Enter the admin password to unlock the dialog.

Panel Sections

General Settings

Configure high-level display preferences:

  • Theme: Select from six built-in themes or enable auto-cycle.
  • Time Format: Toggle between 12-hour (AM/PM) and 24-hour formats.
  • Pre-Adhan Alert: Set the lead time (in minutes) for an alert before prayer time.
  • Volume: Adjust the master volume for Adhan and Iqamah playback.
  • Admin Password: Change the panel password from the default.
  • Developer Password: Set or change the developer-mode password.

Timing & Iqamah Configuration

Fine-tune prayer alert timing per period:

SettingDescription
iqamah_delaysPer-prayer delay (seconds) between Adhan audio and the Iqamah countdown overlay.
adhan_image_delayDuration (seconds) the full-screen Adhan GIF is shown.
iqamah_entry_delayDelay (seconds) after the Adhan GIF hides before the Iqamah overlay fades in.
final_iqamah_delayDuration (seconds) the "SOLAT" glow screen displays after the countdown reaches zero.

Slideshow Configuration

Control the image rotation behavior:

  • Interval: Seconds between image transitions.
  • Effect: Choose between fade, slide, or none.

JAKIM API & Zone

Manage the data source for prayer times:

  • Zone: The Malaysian e-Solat zone code (e.g., PNG01 for Perak).
  • API Endpoint: The JAKIM e-Solat API URL. Developer mode allows direct editing.

If times seem incorrect, verify the zone and re-fetch the annual data.

License & Device Binding

The application locks to the Windows hardware UUID of the machine it first runs on. Administrators can:

  • View the currently bound UUID and device name.
  • Unbind the current device (requires admin authorization).
  • Bind to a new device by entering or selecting its UUID.

Testing & Simulation

The Admin Panel includes a time simulation engine that lets administrators test the entire alert flow without waiting for real time:

  1. Enter a target prayer time or select a pre-canned scenario.
  2. Click Simulate to offset the internal clock.
  3. Observe the countdown, Adhan GIF, Iqamah overlay, and SOLAT glow screens.

This is useful for verifying speaker wiring, volume levels, and animation timing before the live system goes into daily production.

💡 Tip: Use the built-in Test Adhan and Test Iqamah buttons to quickly verify audio output and speaker configuration.

Developer Mode

A password-protected Developer Mode unlocks advanced capabilities:

  • Direct editing of the API endpoint URL.
  • Manual trigger for fetching and rebuilding annual prayer times.
  • Advanced diagnostics and internal state inspection.

Developer mode is intended for troubleshooting and development purposes only.
